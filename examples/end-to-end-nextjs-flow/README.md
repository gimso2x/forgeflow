# Next.js Dashboard Empty State Example

This directory contains representative ForgeFlow artifacts for a common development task: adding an empty-state card to a dashboard.

## Scenario
A Next.js dashboard that displays a list of projects. When no projects exist, it should show an accessible, styled empty-state card with a call-to-action link to create a new project.

## Task ID
`dashboard-empty-state`

## Artifacts
These artifacts were generated during a hypothetical `/forgeflow` run:

- `brief.json`: The clarified objective, scope, acceptance criteria, and selected route (`medium`).
- `plan.json`: The executable implementation steps and verification plan.
- `plan-ledger.json`: The task ledger that maps plan steps to status, gates, and evidence refs.
- `run-state.json`: The execution ledger showing completed steps, gates, and evidence references.
- `decision-log.json`: A record of key implementation decisions and their rationales.
- `review-report.json`: The independent quality review pass with verdict and evidence citations.

## Usage
These files serve as a reference for:
- **Schema validation**: Use them to test that your ForgeFlow integration produces valid JSON.
- **Style guide**: See how to write concise but high-signal objectives, decisions, and review findings.
- **Workflow understanding**: Trace how a task moves from clarify to ship through these documents.
