# Decisions

This directory records durable decisions that affect future ForgeFlow work.

Use it for decisions that would otherwise become tribal knowledge:

- adopting, adapting, or rejecting external workflow ideas;
- changing review, validation, adapter, or artifact contracts;
- drawing a source-repo vs installed-project boundary;
- explaining why a tempting command, stage, gate, schema lane, or runtime state lane was not added.

Keep entries short and boring:

- context;
- decision;
- adopted/adapted/rejected parts;
- consequences;
- validation or follow-up.

Decision records are not runtime state. Task-local execution state still belongs under `.forgeflow/tasks/*` artifacts.
